★ ghss

GitHub Settings Sync

Marca un repositorio como referencia y propaga su configuración a tantos repos como quieras. Audita las diferencias primero, elige qué se sincroniza y aplícalo — sin tocar cada repo a mano.

Instalar Código en GitHub

macOS (Apple Silicon & Intel) · Linux · construida con Tauri 2 + Rust

Qué hace

Administrar decenas de repos a mano es tedioso y propenso a errores: cada repo tiene su pestaña de Settings con docenas de opciones. ghss lee la configuración de un repo que eliges como fuente de verdad y te deja replicarla en un listado de repos destino, mostrándote exactamente qué va a cambiar en cada uno antes de aplicar nada.

Flujo en 4 pasos

  1. Conecta con GitHub (gh CLI, token personal o device flow).
  2. Elige repos: marca uno como referencia ★ y selecciona los destinos (filtra por organización, equipo o búsqueda).
  3. Audita: ghss compara cada destino con la referencia y te muestra las diferencias en streaming.
  4. Sincroniza: desmarca lo que no quieras propagar y aplica. Los fallos de un repo no detienen al resto.

Qué configuración sincroniza

Default branch

La rama por defecto del repositorio (si existe en el destino).

Features

Wikis, Issues, Projects, Discussions y «Allow forking».

Pull Requests

Merge/squash/rebase, mensajes de commit, auto-merge, sugerir actualización de ramas y borrado automático de ramas.

Others

Requerir firma (sign-off) en commits desde la web.

Tags

Rulesets con destino tag (protección de tags moderna).

Rules

Rulesets de rama + branch protection clásica de las ramas protegidas.

Diseñada para listas grandes

  • Selección granular: deselecciona settings concretos por repo antes de sincronizar; todo marcado por defecto, con «Marcar / Desmarcar todo».
  • Auditoría en streaming: los resultados aparecen conforme se procesan, con barra de progreso.
  • Filtros que persisten: búsqueda, organización y equipo, sin perderse al navegar.
  • Robusto ante errores: un repo privado en plan free, archivado o sin permisos no rompe el lote — se marca como fallido con un mensaje claro y el resto continúa.
  • Repos archivados ocultos por defecto y no seleccionables como destino (sí como referencia).

Instalación

Homebrew (macOS)

brew install --cask theburrowhub/tap/ghss

Actualizar: brew upgrade --cask ghss

Descarga directa

Descarga el .dmg de la última release, ábrelo y arrastra ghss a Aplicaciones.

Desde el código

git clone https://github.com/theburrowhub/ghss
cd ghss
npm install
npm run tauri build

Autenticación

Elige el método que prefieras al abrir la app:

  • GitHub CLI (por defecto): reutiliza tu sesión de gh auth login.
  • Token personal (PAT): pégalo y se guarda en el llavero del sistema. Necesita el scope repo para sincronizar repos privados.
  • OAuth Device Flow: introduce un código en github.com (requiere el Client ID de tu propia OAuth App).

Limitaciones conocidas

Algunos toggles de la UI de GitHub no tienen API pública y no se sincronizan: Sponsorships, restringir edición de wikis a colaboradores, comentarios en commits individuales, autocierre de issues con PRs enlazados, límite de actualizaciones por push, incluir objetos Git LFS en archivos, y «PR creation allowed by».

Privacidad

ghss es una app local: tu token nunca sale de tu equipo (se guarda en el llavero del sistema) y todas las llamadas van directas a la API de GitHub. No hay servidores intermedios ni telemetría.
